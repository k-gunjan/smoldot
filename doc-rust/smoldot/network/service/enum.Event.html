Event generated by ChainNetwork::next_event.

Variants

Connected(PeerId)

Established a transport-level connection (e.g. a TCP socket) with the given peer.

Disconnected

Fields

peer_id: PeerId
chain_indices: Vec<usize>

A transport-level connection (e.g. a TCP socket) has been closed.

This event is called unconditionally when a connection with the given peer has been closed. If chain_indices isn’t empty, this event is also equivalent to one or more Event::ChainDisconnected events.

InboundSlotAssigned

Fields

chain_index: usize
peer_id: PeerId

The given peer has opened a block announces substream with the local node, and an inbound slot has been assigned locally to this peer.

A Event::ChainConnected or Event::ChainConnectAttemptFailed will later be generated for this peer.

BlockAnnounce

Fields

peer_id: PeerId

Identity of the sender of the block announce.

chain_index: usize

Index of the chain the block relates to.

announce: EncodedBlockAnnounce

Received a new block announce from a peer.

Can only happen after a Event::ChainConnected with the given PeerId and chain index combination has happened.

ProtocolError

Fields

peer_id: PeerId

Peer that has caused the protocol error.

error: ProtocolError

Error that happened.

Error in the protocol in a connection, such as failure to decode a message. This event doesn’t have any consequence on the health of the connection, and is purely for diagnostic purposes.

Identifier of the request. Necessary to send back the answer.

A remote has sent a request for blocks.

Can only happen for chains where ChainConfig::allow_inbound_block_requests is true.

You are strongly encouraged to call ChainNetwork::respond_blocks.
